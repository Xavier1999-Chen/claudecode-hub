## 2026-04-21 20:09:11 · 无热账号时迁移 vs 不迁移

### Trigger
PR #18 设计 `reassignCoolingTerminals` 时，需决定全员都冷却时怎么办。

### Options Considered
- A · 不迁移（保持原绑定）
  - + 避免把所有终端堆到同一个"least bad"冷却账号上
  - + 用户已经在等了，迁不迁结果一样
  - - UI 上显示的还是原冷却账号
- B · 迁到 resetAt 最早的冷却账号
  - + UI 看起来"在动"
  - - 所有终端集中绑到一个账号，加重该账号恢复时的瞬时冲击
  - - 反复来回切换徒增写盘次数
- C · 让前端显示"全部冷却"提示
  - + UX 最清晰
  - - 需要前端配合，本 PR 范围扩大

### Decision
选 A。proxy 路由层每次请求重新选 cooling 账号（已实现），UI 显示的"原冷却账号"在用户视觉里只是"还没好"，不会误导。集中绑反而有副作用，得不偿失。

# PRD · 营销首页

## 1. 背景与目标

claudecode-hub 目前只有 admin 面板，没有面向陌生访客的入口页。访客无法快速判断这是什么、为什么用、怎么开始用。

**核心定位**：claudecode-hub 是给 Claude Code 用户的一个"既能预算可控、又能按量付费"的共享账号池服务。它解决三个问题：

1. **用量自评困难** —— 用户不知道自己属于轻度/中度/重度，不敢承诺订阅
2. **订阅 vs 按量二选一** —— 自购订阅有预算可控但用不上；按量付费有弹性但开销不可控
3. **自购订阅有封号风险** —— 多设备共享 Pro 账号容易被 Anthropic 风控

**hub 的方案**：
- 共享账号池（无需注册 Anthropic 账号、不用自己买 Pro）
- 阶梯月结（按 30 天周期实测使用强度收费）
- 实时使用强度可视化（用户随时能看到自己处于哪个档位）
- hub 后台自动调度，规避封号

**目标**：让 Claude Code 用户访问首页 1-2 分钟内：
- 理解上面的核心定位
- 找到注册入口完成注册
- 了解大致的产品形态（有 admin 面板、有用量面板）

## 2. 用户故事

### 2.1 已知 Claude Code、找替代订阅方案的开发者（核心场景）
**故事**：作为已经在用 Claude Code 但纠结要不要买 Pro 订阅的开发者，我希望访问首页就能立刻看到 hub 是不是我要的"中间方案"，并完成注册。
**验收标准**：
- [ ] hero 区文案能让用户在不滚动的情况下读懂核心定位
- [ ] hero 区有明确的注册 CTA
- [ ] 滚到下方能看到具体功能（共享账号 / 实时强度 / 阶梯月结 / 免封号）
- [ ] 滚到底部能看到适用人群描述（自我筛选）

### 2.2 已经被 Anthropic 限流过的重度用户
**故事**：作为受过 Pro 限流之苦的开发者，我希望首页能直接讲清楚 hub 怎么解决限流问题（多账号自动调度），而不是让我自己去推导。
**验收标准**：
- [ ] 功能介绍区有"自动调度 / 切换账号"的描述
- [ ] 描述与 admin 面板里实际看到的能力一致（不夸大）

### 2.3 内测期访客（当前阶段）
**故事**：作为内测期访客，我注册后理解需要等待 admin 审批，不会因为"注册不成功"而流失。
**验收标准**：
- [ ] 注册按钮文案 / 注册成功后的页面明确告知"内测期需要 admin 审批"

### 2.4 公测期访客（payment epic 上线后）
**故事**：作为公测期访客，注册即可使用，无需等待审批；进站时余额 = 0，但即使余额不足首页也能让我先看自己未来的强度区间分布。
**验收标准**：
- [ ] payment 上线后，注册后的"等待审批"提示自动消失
- [ ] 文案/CTA 切换到"立即开始"
- [ ] 不需要为公测期改 hero 视觉（只切微文案）

## 3. 功能需求（粗粒度）

> 颗粒度先到"页面有几个段、每段大致放什么"。细化（数据字段、交互、边界）放第四阶段。

### 3.1 页面结构（自上而下）

1. **导航栏（Nav）**
   - 左：Logo
   - 中：锚点链接「定价」（跳到 §3.1.5 定价段）+「适用人群」（跳到 §3.1.4 适用人群段）+「注册」（跳到底部 CTA 段的注册按钮）
   - 右：根据登录态切换：
     - **未登录** → 「登录」按钮（点击跳 `/login`）
     - **已登录** → 「控制台」按钮（点击跳到 admin 主页面）
2. **Hero 区**
   - 主标题：**"共享账号池，让 Claude Code 既好用又划算"**
   - 副标题：**"Pro/Max 共享账号池 · 阶梯月结 · 免封号 · 先用后付"**
   - 主 CTA 按钮 + 次 CTA（如"看看怎么用"，跳到适用人群段）
   - 视觉风格参照 Anthropic 官方首页（有手感、暖色、留白充足；不放具体产品截图）
3. **功能介绍区**（What it does）
   - 4 个功能卡片：
     - **共享账号池** —— 多个 Pro/Max 账号任选择，无需自己注册或购买
     - **实时使用强度** —— 仪表盘随用随动，永远知道自己处于哪一档
     - **阶梯月结** —— 按 30 天周期实测强度结算，4 档透明定价无惊吓
     - **自动调度免封号** —— hub 后台自动切换账号 + 限流回避，让你专注写代码不操心账号风险
4. **适用人群区**（Who it's for）
   - 4 个**个人用户**画像，与定价 4 档对齐（轻度 / 中度 / 重度 / 超重度）
   - 不包含团队/组织角色（首版面向个人用户）
   - **每个画像旁配实时强度计（IntensityGauge）动画**：复用 admin 面板已有的 `IntensityGauge` 组件，针停留在该画像对应档（轻 / 中 / 重 / 超），并在该档区间内做循环动画扫过（让用户直观感受"哦，hub 真的有个实时表盘在跟着我用量动"）。这是页面的核心 hook —— 用图替代解释，**让"实时使用强度"这个抽象卖点变成可以一眼看到的产品形态**。
5. **定价区**（Pricing）
   - 如实展示真实阶梯计费规则
   - 4 档强度（轻度 / 中度 / 重度 / 超重度）+ 各档月结金额（人民币）+ 30 天周期说明
   - **用户视角不展示美金**：以"使用强度"为定价单位（模糊但实时可感知），**不**暴露后台美金阈值
   - 用户视角的价格表：
     - 轻度 → ¥20
     - 中度 → ¥150
     - 重度 → ¥900
     - 超重度 → ¥1500
   - 后台实现细节（不展示给用户）：按累计等价美金用量识别档位（详见 `usage-intensity.js` 的 TIER_THRESHOLDS）
6. **底部 CTA 区**
   - 重申主张 + 注册按钮
7. **Footer**
   - 极简：仅版权（"© 2026 claudecode-hub"），无邮箱、无链接组、无联系方式

## 5. 边界情况与异常处理（用户视角）

### 5.1 路由与登录态
- **已登录用户访问 `/`**：展示完整营销页（不自动跳转），但 Nav 右上角按钮从「登录」切换为「控制台」
- **未登录访客访问 `/`**：展示完整营销页，Nav 右上角是「登录」
- **点「登录」按钮**（未登录态）→ 跳到 `/login` 页
- **点「控制台」按钮**（已登录态）→ 跳到 admin 主页面
- **注册 CTA 的可见性**：
  - 未登录 → 显示注册 CTA（Nav 锚点 + Hero 主 CTA + 底部 CTA），点击跳 `/register`
  - **已登录 → 全部隐藏注册 CTA**（Nav 中的「注册」锚点、Hero 主 CTA、底部 CTA 均不显示）
  - 注册成功的访客落到 pending 等审批（由 `auth-onboarding` epic 处理）

> 实现细节：admin 与 marketing 都用 `@supabase/ssr` —— admin 端用
> `createBrowserClient`（document.cookie 存储），marketing 端用
> `createServerClient`（cookies API 读取）。cookies 不区分端口，所以
> 同 host 的 :3182 和 :3183 共享 session；生产环境配 cookie domain 到
> 父域（如 `.example.com`）即可让子域共享。issue #59 在本 Story 内一并解决。

### 5.2 锚点行为
- 点导航栏「定价」/「适用人群」/「注册」→ 平滑滚动到对应段（`scrollIntoView({behavior:'smooth'})`）
- URL hash（如 `/#pricing`）支持直接定位，分享链接到具体段可用
- 浏览器后退按钮按 hash history 行为返回上一锚点

### 5.3 强度计动画降级
- 强度计动画 4 个并排播放可能在低端机上卡顿
- **降级策略**：监测 `prefers-reduced-motion: reduce` 媒体查询 → 显示静态终态（针停在该档区间末，不动）
- 用户主动开启系统"减少动画"偏好时同上

### 5.4 内容可访问性
- **JS 禁用情况**：不保证（受众均为开发者，禁 JS 概率极低；引入 SSR/prerender 与本 Story 工程量不成比例）
- 所有按钮 / 链接有可读 `aria-label`（屏幕阅读器友好的基础水平）

### 5.5 浏览器与设备
- **桌面端**：Chrome / Edge / Safari / Firefox 最新两个版本
- **移动端**：不主动适配（已 OOS）；最低保证不报错（即使布局错位也能阅读文字）
- **最小桌面宽度**：1280px（窄于此可能横向滚动，不保证视觉完美）

### 5.6 注册流的边界
- 注册流本身的异常（邮箱已存在 / Supabase 不可达 / 邮件发不出）由 `auth-onboarding` epic 处理，营销页不重复
- 营销页只保证「点 CTA → 进 `/register`」这一跳转链路可靠
